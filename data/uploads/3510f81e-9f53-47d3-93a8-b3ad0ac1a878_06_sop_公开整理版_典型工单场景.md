# SOP：公开整理版典型工单场景

## 场景 1：材料不全，审核暂停

### 场景描述

被保险人完成住院后，提交了理赔申请，但只上传了发票，没有上传完整病历和费用清单。

### 处理目标

- 补齐支持责任判断和费用核算的材料
- 让案件从“暂停审核”回到“继续审核”

### 标准处理步骤

1. 核对当前已提交材料
2. 识别缺失项：出院小结、诊断证明、费用清单等
3. 通过官方渠道通知补件
4. 申请人回医院补打费用清单、补取病历或检查报告
5. 再次提交材料
6. 进入复审

### 知识点

- 发票证明金额
- 病历证明治疗事实
- 费用清单证明项目构成

## 场景 2：发票缺失或由第三方先行赔付

### 场景描述

被保险人先经过医保或第三方结算，自己手里没有完整原始票据，只保留了部分复印件或结算记录。

### 处理目标

- 补齐分割单、结算单或对应票据证明
- 让审核能够判断实际自付和可赔范围

### 标准处理步骤

1. 确认是否已存在医保或第三方先行赔付
2. 获取分割单或结算清单
3. 对应补充票据复印件或电子票据
4. 说明原始票据去向
5. 提交材料给保险公司复核

### 知识点

- 理赔审核需要看实际费用承担情况
- 第三方结算场景下，票据和分割单往往是成对使用

## 场景 3：在非约定医院急诊就诊

### 场景描述

被保险人因为情况紧急，先在就近医院急诊治疗，后续才考虑转院或补报案。

### 处理目标

- 说明急诊原因
- 尽快补通知
- 解释非约定医院就诊的合理性

### 标准处理步骤

1. 保留急诊病历和入院记录
2. 尽快通知保险公司
3. 说明为何无法直接前往约定医院
4. 病情稳定后按要求转入约定医院或完成后续说明
5. 提交完整病历链条

### 知识点

- 紧急情形通常是例外处理高发点
- 是否及时说明并保留证据，会直接影响后续审核判断

## 场景 4：健康告知和既往症被重新核查

### 场景描述

被保险人住院后申请理赔，审核过程中保险公司回看投保时的健康告知，发现既往病史、过往检查或就诊记录需要进一步核实。

### 处理目标

- 判断本次理赔是否与既往病史、等待期或除外责任有关
- 澄清信息是否一致

### 标准处理步骤

1. 核对本次住院诊断与既往病史
2. 核对投保时的健康告知内容
3. 如被要求，补充历史病历或检查资料
4. 结合条款进行责任判断
5. 输出审核结论

### 知识点

- 健康告知真实与否会影响理赔结论
- 不是所有病史都等于拒赔，但一定可能影响审核

## 场景 5：案件已结案，但申请人未收到赔款

### 场景描述

系统显示案件已结案，但申请人反馈还没有到账。

### 处理目标

- 判断问题出在结案后支付、账户信息还是尾部材料

### 标准处理步骤

1. 确认是否为“审核结案”还是“支付完成”
2. 核对收款账户信息
3. 核对是否还有原件或补充材料未完成
4. 确认支付渠道状态
5. 继续跟进到账结果

### 知识点

- 结案、支付、到账是三个不同节点
- 流程型知识库里要把这三层拆开

## 场景 6：住院费用里含明显不在常规责任范围内的项目

### 场景描述

申请材料中同时包含住院治疗费用和部分明显边界性项目，例如非医学必要项目、康复器械或其他特殊费用。

### 处理目标

- 区分标准住院医疗费用和边界项目
- 让审核结论从“全赔/全拒”变成更接近真实情况的“责任拆分”

### 标准处理步骤

1. 查看费用明细清单
2. 标注边界性项目
3. 回到条款和责任说明判断
4. 对属于责任范围内的费用继续核算
5. 对非责任范围项目作出说明

### 知识点

- 费用型医疗险强调责任边界和补偿原则
